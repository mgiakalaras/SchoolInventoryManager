# School Inventory Manager

Web εφαρμογή για απογραφή, τεχνική παρακολούθηση και διαχείριση σχολικού ψηφιακού εξοπλισμού.

Η εφαρμογή βοηθά τον υπεύθυνο απογραφής / Πληροφορικής να οργανώνει χώρους, εξοπλισμό, QR ετικέτες, φακέλους απογραφής, τεχνικά χαρακτηριστικά, ανταλλακτικά, αναφορές και διαδικασίες καταστροφής υλικού μέσα στο τοπικό σχολικό δίκτυο.

> Τρέχουσα φάση: **School testing**  
> Προτεινόμενη χρήση: **τοπικό σχολικό δίκτυο / VPN**  
> Μη προτεινόμενη χρήση: **δημόσια έκθεση στο Internet χωρίς login / HTTPS / roles**

## Βασική αρχιτεκτονική ροής QR

Η QR απογραφή χωρίζεται ξεκάθαρα σε δύο ρόλους:

### Web app

Το web app είναι το κέντρο διαχείρισης.

Χρησιμοποιείται για:

- διαχείριση χώρων
- διαχείριση εξοπλισμού
- δημιουργία φακέλων απογραφής
- προβολή προόδου απογραφής
- χειροκίνητο έλεγχο / διόρθωση όπου χρειάζεται
- εκτύπωση QR ετικετών
- αναφορές και οριστικοποιήσεις

Το web app **δεν είναι πλέον η κύρια ροή camera scanning**. Από υπολογιστή χρησιμοποιείται κυρίως για διαχείριση και χειροκίνητη καταχώρηση.

### Android Scanner

Η πραγματική σάρωση QR με κάμερα γίνεται από την εφαρμογή Android Scanner.

Η Android εφαρμογή:

- συνδέεται στο web app API
- φορτώνει φακέλους απογραφής
- φορτώνει χώρους φακέλου
- σκανάρει QR με κάμερα
- υποστηρίζει χειροκίνητη καταχώρηση κωδικού ως fallback
- στέλνει τα scans στον server
- ενημερώνει live την πρόοδο του φακέλου / χώρου

Ο server παραμένει η μοναδική πηγή αλήθειας για τα αποτελέσματα απογραφής.

## QR κωδικοί και ετικέτες

### Μοναδικός κωδικός απογραφής

Κάθε αντικείμενο αποκτά μοναδικό κωδικό τύπου:

```text
SIM-2026-000123
```

Ο κωδικός δεν πρέπει να αλλάζει, γιατί χρησιμοποιείται από τα QR labels και τις μελλοντικές απογραφές.

### QR read-only καρτέλα αντικειμένου

Κάθε QR οδηγεί σε καθαρή καρτέλα προβολής:

```text
/q/SIM-2026-000123
```

Η σύντομη διαδρομή `/q/{code}` κάνει redirect στην καρτέλα αντικειμένου και κρατά το QR απλό ώστε να διαβάζεται πιο εύκολα.

Η καρτέλα εμφανίζει:

- ονομασία
- κωδικό απογραφής
- χώρο
- κατηγορία
- κατάσταση
- serial number
- μάρκα / μοντέλο
- ποσότητα
- τεχνικά χαρακτηριστικά, όπου υπάρχουν

### QR labels

Υπάρχει σελίδα:

```text
/Labels/Qr
```

Υποστηρίζει:

- λίστα αντικειμένων
- αναζήτηση
- φίλτρο χώρου
- φίλτρο κατηγορίας
- pagination
- επιλογή συγκεκριμένων αντικειμένων
- εκτύπωση QR ετικετών
- διάταξη για Typotrust TL2111 labels:
  - A4
  - 21 ετικέτες ανά φύλλο
  - 3 στήλες x 7 σειρές
  - 70mm x 42.3mm

## Φάκελοι απογραφής

Η επίσημη απογραφή γίνεται μέσα από φάκελο απογραφής.

### Ροή εργασίας

1. Δημιουργία φακέλου απογραφής
2. Αυτόματη λήψη στοιχείων σχολείου από Settings
3. Δημιουργία γραμμών απογραφής ανά χώρο
4. Άνοιγμα φακέλου / χώρου από Android Scanner
5. Σάρωση QR αντικειμένων με κινητό ή tablet
6. Live ενημέρωση προόδου στον server
7. Έλεγχος βρεθέντων / ελλειπόντων / λάθος χώρου / άγνωστων QR
8. Χειροκίνητες διορθώσεις από web app όπου χρειάζεται
9. Οριστικοποίηση χώρου
10. Οριστικοποίηση φακέλου

### Τι αποθηκεύει ο φάκελος

Ο φάκελος κρατά snapshot των στοιχείων σχολείου κατά τη δημιουργία:

- σχολική μονάδα
- τύπος σχολείου
- σχολικό έτος
- ημερομηνία απογραφής
- υπεύθυνος/η απογραφής

Αυτό είναι σημαντικό ώστε οι παλιοί φάκελοι να κρατούν τα στοιχεία που ίσχυαν όταν δημιουργήθηκαν.

## Docker / Portainer deployment

Η εφαρμογή μπορεί να τρέξει σε σχολικό server, ZimaOS, CasaOS, Linux server ή Windows μηχάνημα με Docker.

### Docker Compose

```bash
docker compose up -d --build
```

Άνοιγμα από άλλη συσκευή στο ίδιο δίκτυο:

```text
http://SERVER-IP:5148
```

Παράδειγμα:

```text
http://192.168.1.80:5148
```

### Portainer

Στο Portainer:

```text
Stacks → Add stack → Repository
```

Ρυθμίσεις:

```text
Name: school-inventory-manager
Repository URL: https://github.com/mgiakalaras/SchoolInventoryManager.git
Repository reference: refs/heads/main
Compose path: docker-compose.yml
```

Η εφαρμογή χρησιμοποιεί Docker named volume για τα δεδομένα:

```text
school_inventory_app_data
```

## Ρύθμιση QR για κινητά / tablets

Στα Settings πρέπει να οριστεί η διεύθυνση εφαρμογής στο τοπικό δίκτυο.

Παράδειγμα:

```text
http://192.168.1.80:5148
```

Αν τα QR δείχνουν σε `localhost`, τότε δεν θα ανοίγουν από κινητό / tablet.

Για να δουλέψει η Android εφαρμογή:

- το κινητό / tablet πρέπει να βλέπει τον server στο ίδιο δίκτυο ή μέσω VPN
- το `ApplicationBaseUrl` στα Settings πρέπει να δείχνει στον server
- το web app πρέπει να είναι προσβάσιμο από το κινητό
- ο φάκελος απογραφής πρέπει να έχει δημιουργηθεί μετά την εισαγωγή / ενημέρωση των αντικειμένων
