# School Inventory Manager

Web εφαρμογή για την απογραφή, τεχνική παρακολούθηση και διαχείριση ψηφιακού εξοπλισμού σε σχολικές μονάδες.

Η εφαρμογή βοηθά τον υπεύθυνο απογραφής / Πληροφορικής να καταχωρίζει χώρους, εξοπλισμό, τεχνικά χαρακτηριστικά, ανταλλακτικά, αναλώσιμα, αναφορές, καταστροφές υλικού, QR ετικέτες και διαδικασίες απογραφής χώρων από υπολογιστή, tablet ή κινητό μέσα στο τοπικό σχολικό δίκτυο.

> Τρέχουσα φάση: **School testing**  
> Προτεινόμενη χρήση: **τοπικό σχολικό δίκτυο**  
> Μη προτεινόμενη χρήση: **δημόσια έκθεση στο Internet χωρίς login / HTTPS / roles**

---

## Τι κάνει η εφαρμογή

Το School Inventory Manager ξεκίνησε ως απλή εφαρμογή απογραφής σχολικού εξοπλισμού και εξελίσσεται σε εργαλείο τεχνικής διαχείρισης σχολικής υποδομής.

Υποστηρίζει:

- χώρους σχολείου
- κατηγορίες εξοπλισμού
- καταχώριση υλικών ανά χώρο
- κατάσταση εξοπλισμού
- Excel import / export
- duplicate review κατά την εισαγωγή
- αναφορές απογραφής
- εκτύπωση / Save as PDF
- διαδικασία καταστροφής υλικού
- τεχνικά χαρακτηριστικά Η/Υ / laptop / mini PC / διαδραστικών
- βάση τεχνικών αναφορών για CPU, RAM, δίσκους, λειτουργικά κ.λπ.
- απόθεμα ανταλλακτικών και αναλώσιμων
- ιστορικό χρήσης ανταλλακτικών
- τεχνικό έλεγχο και υποψήφιες αναβαθμίσεις
- QR labels ανά αντικείμενο
- QR read-only καρτέλα αντικειμένου
- σάρωση QR / έλεγχο ύπαρξης αντικειμένου
- φάκελο απογραφής
- απογραφή χώρου με QR
- αποθήκευση προόδου απογραφής ανά χώρο
- οριστικοποίηση χώρου και φακέλου απογραφής
- dashboard τύπου control center

## QR απογραφή

### Μοναδικός κωδικός απογραφής

Κάθε αντικείμενο αποκτά μοναδικό κωδικό τύπου:

```text
SIM-2026-000123
```

Ο κωδικός δεν πρέπει να αλλάζει, γιατί χρησιμοποιείται από τα QR labels και τις μελλοντικές απογραφές.

### QR read-only καρτέλα αντικειμένου

Κάθε QR οδηγεί σε καθαρή καρτέλα προβολής:

```text
/q/SIM-2026-000123
```

Η σύντομη διαδρομή `/q/{code}` κάνει redirect στην καρτέλα αντικειμένου και κρατά το QR πιο απλό ώστε να διαβάζεται πιο εύκολα από κινητό.

Η καρτέλα εμφανίζει:

- ονομασία
- κωδικό απογραφής
- χώρο
- κατηγορία
- κατάσταση
- serial number
- μάρκα / μοντέλο
- ποσότητα
- τεχνικά χαρακτηριστικά, όπου υπάρχουν

### QR labels

Υπάρχει σελίδα:

```text
/Labels/Qr
```

Υποστηρίζει:

- λίστα αντικειμένων
- αναζήτηση
- φίλτρο χώρου
- φίλτρο κατηγορίας
- pagination
- επιλογή συγκεκριμένων αντικειμένων
- εκτύπωση QR ετικετών
- διάταξη για Typotrust TL2111 labels:
  - A4
  - 21 ετικέτες ανά φύλλο
  - 3 στήλες x 7 σειρές
  - 70mm x 42.3mm

### QR scanner

Υπάρχει σελίδα:

```text
/Audit/Scan
```

Υποστηρίζει:

- χειροκίνητο έλεγχο κωδικού
- έλεγχο πλήρους QR URL
- προβολή αν το αντικείμενο υπάρχει στην απογραφή
- πρόσφατες σαρώσεις
- camera scan όταν το υποστηρίζει ο browser

Σημείωση: Η κάμερα μέσα από browser σε κινητό συνήθως απαιτεί HTTPS. Σε απλό HTTP τοπικού δικτύου, όπως `http://192.168.1.80:5148`, μπορεί να μη δοθεί πρόσβαση στην κάμερα από τον browser. Σε αυτή την περίπτωση δουλεύει κανονικά η χειροκίνητη αναζήτηση ή το εξωτερικό QR scanner/camera app του κινητού που ανοίγει το URL.

## Φάκελοι απογραφής και QR απογραφή χώρου

Η εφαρμογή υποστηρίζει πλέον επίσημη ροή απογραφής με φάκελο.

### Ροή εργασίας

1. Δημιουργία φακέλου απογραφής
2. Αυτόματη λήψη στοιχείων σχολείου από Settings
3. Δημιουργία γραμμών απογραφής ανά χώρο
4. Άνοιγμα χώρου
5. Σάρωση QR αντικειμένων
6. Αποθήκευση προόδου
7. Έλεγχος βρεθέντων / ελλειπόντων / λάθος χώρου / άγνωστων QR
8. Οριστικοποίηση χώρου
9. Οριστικοποίηση φακέλου

### Φάκελος απογραφής

Ο φάκελος κρατά snapshot των στοιχείων σχολείου κατά τη δημιουργία:

- σχολική μονάδα
- τύπος σχολείου
- σχολικό έτος
- ημερομηνία απογραφής
- υπεύθυνος/η απογραφής

Αυτό είναι σημαντικό ώστε οι παλιοί φάκελοι να κρατούν τα στοιχεία που ίσχυαν όταν δημιουργήθηκαν.

### Απογραφή χώρου με QR

Υπάρχει σελίδα:

```text
/Audit/Room
```

Όταν ανοίγει μέσα από φάκελο απογραφής:

- φορτώνει τα αναμενόμενα αντικείμενα του χώρου
- αποθηκεύει κάθε σάρωση
- ξαναφορτώνει την πρόοδο μετά από refresh
- ενημερώνει μετρητές:
  - αναμενόμενα
  - βρέθηκαν
  - λείπουν
  - άλλου χώρου / άγνωστα
- επιτρέπει οριστικοποίηση χώρου
- κλειδώνει οριστικοποιημένο χώρο
- επιτρέπει άνοιγμα για διορθώσεις όσο ο φάκελος δεν έχει οριστικοποιηθεί

- χώρο

- μοντέλο

Αν μια εγγραφή μοιάζει με ήδη υπάρχουσα, ο χρήστης μπορεί να αποφασίσει αν θα την παραλείψει, θα ενημερώσει την παλιά εγγραφή ή θα την περάσει ως νέα.

- CSV export

## Docker / Portainer deployment

Η εφαρμογή μπορεί να τρέξει σε σχολικό server, ZimaOS, CasaOS, Linux server ή Windows μηχάνημα με Docker.

### Docker Compose

```bash
docker compose up -d --build
```

Άνοιγμα από άλλη συσκευή στο ίδιο δίκτυο:

```text
http://SERVER-IP:5148
```

Παράδειγμα:

```text
http://192.168.1.80:5148
```

### Portainer

Στο Portainer:

```text
Stacks → Add stack → Repository
```

Ρυθμίσεις:

```text
Name:
school-inventory-manager

Repository URL:
https://github.com/mgiakalaras/SchoolInventoryManager.git

Repository reference:
refs/heads/main

Compose path:
docker-compose.yml
```

Η εφαρμογή χρησιμοποιεί Docker named volume για τα δεδομένα:

```text
school_inventory_app_data
```

## Ρύθμιση QR για κινητά / tablets

Στα Settings πρέπει να οριστεί η διεύθυνση εφαρμογής στο τοπικό δίκτυο.

Παράδειγμα:

```text
http://192.168.1.80:5148
```

Αν τα QR δείχνουν σε `localhost`, τότε δεν θα ανοίγουν από κινητό/tablet. Το κινητό πρέπει να βλέπει την IP ή το hostname του server στο σχολικό δίκτυο.

Για να δουλέψουν τα QR από κινητό/tablet:

- το κινητό/tablet πρέπει να είναι στο ίδιο δίκτυο ή μέσω VPN/Tailscale
- το `ApplicationBaseUrl` στα Settings πρέπει να δείχνει στον server
- η εφαρμογή πρέπει να είναι προσβάσιμη από το κινητό
- εξωτερικό QR scanner app ή camera app ανοίγει το URL
- in-browser camera scanning μπορεί να χρειαστεί HTTPS
