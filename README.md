# School Inventory Manager

**Ψηφιακή απογραφή σχολικού υλικού · Nyx Systems**

Web εφαρμογή για την απογραφή Η/Υ, tablets, οθονών, εκτυπωτών, βιντεοπροβολέων και λοιπού ψηφιακού εξοπλισμού σε σχολικές μονάδες.

Στόχος είναι ο/η υπεύθυνος/η απογραφής να μπορεί να καταχωρίζει χώρους και εξοπλισμό από υπολογιστή, tablet ή κινητό, να εισάγει/εξάγει δεδομένα από Excel, να παράγει καθαρές αναφορές απογραφής και να διαχειρίζεται υλικά που είναι μη λειτουργικά, προς απόσυρση ή προς καταστροφή.

> Τρέχουσα έκδοση: **v0.3.0**  
> Τρέχουσα φάση: **Development / School testing**  
> Προτεινόμενη χρήση: **τοπικό σχολικό δίκτυο**  
> Δεν προτείνεται δημόσια έκθεση στο Internet χωρίς authentication, HTTPS και πολιτική backup.

---

## Περιγραφή

Το **School Inventory Manager** ξεκίνησε από πραγματική ανάγκη σχολικής απογραφής και εξελίσσεται σε εργαλείο διαχείρισης σχολικού εξοπλισμού.

Η εφαρμογή βοηθά στη διαχείριση:

- χώρων σχολείου
- κατηγοριών εξοπλισμού
- ενεργών υλικών
- μη λειτουργικών υλικών
- υλικών προς καταστροφή
- ιστορικού κατεστραμμένων / αποσυρμένων υλικών
- αναφορών και εκτυπώσεων
- εισαγωγής/εξαγωγής δεδομένων

Η λογική είναι να τρέχει σε τοπικό σχολικό server ή υπολογιστή και να είναι προσβάσιμη από browser μέσα στο ίδιο δίκτυο.

## Καταστροφή / Απόσυρση υλικού

Από την έκδοση **v0.3.0** προστέθηκε νέο module για τη διαχείριση μη λειτουργικών ή κατεστραμμένων υλικών.

### Υποστηρίζονται:

- επιλογή μη λειτουργικών / κατεστραμμένων υλικών
- δημιουργία φακέλου καταστροφής
- καταχώριση πράξης καταστροφής
- καταχώριση πρωτοκόλλου καταστροφής
- συμπλήρωση μελών επιτροπής
- αναφορά επιλεγμένων υλικών
- εκτύπωση / PDF πράξης καταστροφής
- εκτύπωση / PDF πρωτοκόλλου καταστροφής
- ιστορικό κατεστραμμένων υλικών
- σύνδεση κατεστραμμένων υλικών με τον αντίστοιχο φάκελο καταστροφής

### Σημαντική λογική ασφαλείας

Τα υλικά που καταστρέφονται **δεν διαγράφονται οριστικά** από τη βάση.

Αντί για hard delete:

- αφαιρούνται από την ενεργή απογραφή
- χαρακτηρίζονται ως κατεστραμμένα / αποσυρμένα
- παραμένουν διαθέσιμα στο ιστορικό
- συνδέονται με τον αντίστοιχο φάκελο καταστροφής, πράξη και πρωτόκολλο

Έτσι διατηρείται διοικητικό και ιστορικό ίχνος.

## Docker / Portainer deployment

Η εφαρμογή μπορεί να στηθεί σε σχολικό server, ZimaOS, CasaOS, Linux server ή Windows μηχάνημα με Docker.

### Docker Compose

Από τον φάκελο του project:

```bash
docker compose up -d --build
```

Άνοιγμα από άλλη συσκευή στο ίδιο δίκτυο:

```text
http://SERVER-IP:5148
```

Παράδειγμα:

```text
http://192.168.1.80:5148
```

### Portainer

Στο Portainer:

```text
Stacks → Add stack → Repository
```

Ρυθμίσεις:

```text
Name:
school-inventory-manager

Repository URL:
https://github.com/mgiakalaras/SchoolInventoryManager.git

Repository reference:
refs/heads/main

Compose path:
docker-compose.yml
```

Η εφαρμογή χρησιμοποιεί Docker named volume για τα δεδομένα:

```text
school_inventory_app_data
```

Η βάση SQLite και τα προσωρινά import αρχεία παραμένουν εκεί ώστε να μη χάνονται σε redeploy.
