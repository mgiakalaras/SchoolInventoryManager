# Web QR workflow cleanup

## Απόφαση ροής

Η web εφαρμογή δεν πρέπει να λειτουργεί ως camera QR scanner.

Η επίσημη ροή πλέον είναι:

1. Web app:
   - χώροι
   - εξοπλισμός
   - φάκελοι απογραφής
   - QR ετικέτες
   - αναφορές
   - χειροκίνητη καταχώρηση / έλεγχος όπου χρειάζεται

2. Android Scanner:
   - πραγματική σάρωση QR με κάμερα
   - αποστολή scan στον server
   - live ενημέρωση φακέλου / χώρου

## Τι καθαρίστηκε σε αυτό το patch

- Αφαιρέθηκε από το κεντρικό menu η άμεση επιλογή:
  - `/Audit/Room` — `Απογραφή χώρου QR`

- Το menu `Απογραφή QR` έγινε:
  - `Απογραφή & QR`

- Το menu πλέον δείχνει μόνο:
  - Φάκελοι απογραφής
  - Νέος φάκελος
  - QR ετικέτες
  - Android Scanner
  - Αναφορές απογραφής

## Τι δεν πειράχτηκε ακόμα

- Η σελίδα `/Audit/Room` δεν διαγράφεται.
  Μπορεί ακόμα να χρησιμοποιείται εσωτερικά από φάκελο απογραφής ή για debugging.

- Δεν αφαιρέθηκε ακόμα ο browser-camera κώδικας από `/Audit/Room`.
  Αυτό πρέπει να γίνει σε επόμενο patch αφού επιβεβαιωθεί από πού ανοίγει η επίσημη χειροκίνητη ροή.

## Επόμενα καθαρίσματα repo

1. README:
   - να φύγει ότι γράφει πως το web κάνει camera scan.
   - να γράψει καθαρά ότι QR camera scan γίνεται από Android app.

2. `/Audit/Room`:
   - να γίνει σελίδα “Χειροκίνητη καταχώρηση / έλεγχος χώρου”.
   - να αφαιρεθεί camera box, video element, start/stop camera buttons και browser scanner JS.

3. `/Audit/Scan`:
   - είτε να κρυφτεί
   - είτε να μετονομαστεί σε “Χειροκίνητος έλεγχος QR”
   - είτε να μεταφερθεί σε debug/tools area.

4. `/Scanner/Index`:
   - να γίνει καθαρή σελίδα οδηγιών για εγκατάσταση/χρήση Android Scanner.
