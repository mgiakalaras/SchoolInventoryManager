# School Inventory Manager v0.7.0

## First inventory mobile discovery workflow

Η έκδοση `v0.7.0` κλειδώνει τη νέα ροή **Πρώτη απογραφή / από μηδενική βάση**.

Μέχρι την προηγούμενη φάση, η εφαρμογή ήταν πολύ δυνατή όταν υπήρχε ήδη καταχωρισμένη βάση εξοπλισμού. Με αυτή την έκδοση μπορεί πλέον να χρησιμοποιηθεί και σε σχολείο που ξεκινά απογραφή από το μηδέν: δημιουργία κενού φακέλου, δημιουργία χώρων από κινητό/tablet, καταχώριση νέων ευρημάτων και έλεγχος/QR από το web app.

---

## Κύρια νέα λειτουργικότητα

### Πρώτη απογραφή από μηδενική βάση

Προστέθηκε επιλογή κατά τη δημιουργία φακέλου απογραφής:

```text
Πρώτη απογραφή / από μηδενική βάση
```

Σε αυτή τη ροή ο φάκελος δημιουργείται κενός:

- δεν αντιγράφει υπάρχοντες χώρους
- δεν αντιγράφει υπάρχοντα αντικείμενα
- δεν δημιουργεί room sessions από την υπάρχουσα βάση
- λειτουργεί ως discovery/baseline mode

Ο στόχος είναι να μπορεί ένα σχολείο χωρίς οργανωμένη προηγούμενη απογραφή να χτίσει τη βάση του επιτόπου.

### Ετήσιος έλεγχος υπάρχουσας βάσης

Η προηγούμενη ροή παραμένει διαθέσιμη ως:

```text
Ετήσιος έλεγχος υπάρχουσας βάσης
```

Σε αυτή τη ροή ο φάκελος δημιουργεί room sessions από τους ήδη καταχωρισμένους χώρους και αντικείμενα.

## Mobile / Android workflow

Η εφαρμογή web προσθέτει νέο mobile API endpoint για δημιουργία χώρου μέσα σε φάκελο απογραφής:

```http
POST /api/mobile/audit-folders/{folderId}/rooms/create
```

Αυτό επιτρέπει στο Android Scanner να δημιουργεί νέους χώρους μέσα σε φάκελο πρώτης απογραφής.

Η πλήρης ροή πλέον είναι:

1. Δημιουργία φακέλου πρώτης απογραφής από το web app.
2. Άνοιγμα φακέλου από Android/mobile scanner.
3. Δημιουργία νέου χώρου από Android.
4. Άνοιγμα του νέου χώρου.
5. Προσθήκη νέων αντικειμένων από Android.
6. Review νέων ευρημάτων από web app.
7. Εκτύπωση QR labels για τα νέα ευρήματα.
8. Μελλοντική χρήση της εγκεκριμένης απογραφής ως baseline.

## Διορθώσεις και σταθεροποίηση

### First inventory folder fix

Διορθώθηκε το αρχικό πρόβλημα όπου ο φάκελος πρώτης απογραφής εξακολουθούσε να εμφανίζει υπάρχοντες χώρους/αντικείμενα.

Πλέον ο φάκελος πρώτης απογραφής ξεκινά πραγματικά κενός.

### Mobile API route fix

Διορθώθηκε το route του endpoint δημιουργίας χώρου ώστε να ακολουθεί το υπάρχον mobile API convention:

```text
/api/mobile/audit-folders/...
```

### Schema warning cleanup

Καθαρίστηκαν τα EF1002 warnings στο `Data/SchemaUpgrade.cs` χωρίς αλλαγή λειτουργικής συμπεριφοράς του schema upgrade helper.

### Warning-free release state

Η έκδοση κλειδώνει σε κατάσταση:

```text
0 errors
0 warnings
```

για το web app, μετά τον καθαρισμό των EF1002 warnings.

## Σημειώσεις χρήσης

Η ροή πρώτης απογραφής είναι κατάλληλη για:

- σχολεία που δεν έχουν παλιά απογραφή σε ψηφιακή μορφή
- πρώτη καταγραφή χώρων και εξοπλισμού από tablet/κινητό
- επιτόπια καταγραφή αντικειμένων ανά αίθουσα
- μετέπειτα έλεγχο/διόρθωση από PC
- εκτύπωση QR labels μόνο για τα νέα ευρήματα

Η εφαρμογή παραμένει σε φάση:

```text
Development / School testing
```

και προτείνεται να χρησιμοποιείται σε τοπικό σχολικό δίκτυο.
